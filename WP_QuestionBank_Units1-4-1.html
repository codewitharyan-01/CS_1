Q1 What is HTTP? Explain HTTP Request and Response. ▾

HTTP (HyperText Transfer Protocol) is the foundation of data communication on the World Wide Web. It is an application-layer protocol that defines how messages are formatted and transmitted between a web browser (client) and a web server.

Key Characteristics

  • Stateless: Each request is independent; the server does not remember previous requests.
  • Client-Server Model: The client sends requests; the server responds.
  • Port: HTTP uses port 80 (HTTPS uses port 443).
  • Methods: GET, POST, PUT, DELETE, HEAD, OPTIONS, etc.

HTTP Request Structure

GET /index.html HTTP/1.1
Host: www.example.com
Accept: text/html
Connection: keep-alive
User-Agent: Mozilla/5.0
HTTP

An HTTP request has three parts: Request Line (method + URL + version), Headers (metadata like Host, Accept), and an optional Body (used in POST/PUT).

HTTP Response Structure

HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Content-Length: 1234
Date: Mon, 01 Jan 2024 12:00:00 GMT

<html>
  <body>Hello World</body>
</html>
HTTP

Q2 Briefly discuss: Client Server Architecture. ▾

Client-Server Architecture is a computing model in which tasks are distributed between service providers (servers) and service requesters (clients).

Components

  • Client: Requests services/resources (browser, mobile app). Handles user interface.
  • Server: Provides services/resources. Processes requests and sends responses.
  • Network: The communication medium (Internet/LAN) between them.

Types

TypeDescription
2-TierClient directly communicates with server (e.g., desktop app + DB)
3-TierClient → Application Server → Database Server (most web apps)
N-TierMultiple layers (presentation, business, data, etc.)

Advantages

  • Centralized data management and security
  • Easy scalability — add more clients without changing server
  • Resources are shared efficiently
Q3 Define: URL, HTTP, Web Server ▾

1. URL — Uniform Resource Locator

A URL is the complete address used to identify a specific resource on the internet. It tells the browser where to go and how to get there.

https://www.example.com:8080/path/page.html?id=5#section

Protocol:// Domain:Port/Path?Query#Fragment
  • Protocol: https — how to communicate
  • Domain: www.example.com — which server
  • Port: 8080 — which service (optional)
  • Path: /path/page.html — which resource
  • Query: ?id=5 — additional parameters
  • Fragment: #section — specific section on page

2. HTTP — HyperText Transfer Protocol

HTTP is the application-layer protocol used for transmitting hypermedia documents (HTML). It defines the rules for communication between clients (browsers) and web servers using request-response messaging.

3. Web Server

A web server is software (and/or hardware) that stores, processes, and delivers web pages to clients. It listens for HTTP requests and responds with the appropriate content.

  • Examples: Apache HTTP Server, Nginx, IIS (Microsoft)
  • Serves static files (HTML, CSS, images) directly
  • Can pass dynamic requests to application servers (PHP, Node.js)

Web Design

Q1 Explain points to consider for planning a website and design issues for an effective website. ▾

Planning a Website — Key Considerations

  1. Define Purpose & Goals: What is the website for? (E-commerce, portfolio, blog, informational). Clear goals drive design decisions.
  2. Identify Target Audience: Age, technical literacy, devices used. Design for users, not designers.
  3. Content Planning: What pages are needed? What content goes on each? A content inventory before design prevents gaps.
  4. Site Structure / Information Architecture: How pages relate to each other. Use a hierarchical tree (sitemap) to organize.
  5. Technology Stack: Static HTML? CMS like WordPress? Dynamic PHP/MySQL? Choose based on requirements.
  6. Budget & Timeline: Resources determine scope. Prioritize features accordingly.
  7. Domain & Hosting: Domain name (brand identity) and reliable hosting infrastructure.
  8. SEO Strategy: Plan keywords, meta tags, and URL structure from day one.

Design Issues to Address

  • Consistency: Uniform fonts, colors, button styles across all pages.
  • Responsiveness: Works on desktop, tablet, and mobile screens.
  • Load Speed: Compress images, minimize CSS/JS, use CDN.
  • Accessibility (a11y): Alt text for images, keyboard navigation, contrast ratios.
  • Cross-browser Compatibility: Test on Chrome, Firefox, Safari, Edge.
  • Security: HTTPS, form validation, protect against XSS and SQL injection.
  • Color & Typography: Readable fonts (min 16px body), accessible color contrast (4.5:1 ratio).
  • White Space: Breathing room between elements prevents cognitive overload.
Q2 What is navigation? List the characteristics of effective navigation. ▾

Navigation refers to the system of links, menus, and interface elements that allow users to move between pages and sections of a website. Good navigation is the backbone of usability.

Types of Navigation

  • Primary Navigation: Main menu (usually in header) — links to major sections.
  • Secondary Navigation: Sub-menus, sidebars — links within sections.
  • Footer Navigation: Links to policies, contact, sitemap.
  • Breadcrumb Navigation: Shows user's current path (Home > Products > Shoes).
  • In-page Navigation: Jump links to sections on the same page (anchor links).

Characteristics of Effective Navigation

  1. Clarity: Links should use clear, descriptive labels. "Contact Us" beats "Click Here".
  2. Consistency: Navigation appears in the same location on every page.
  3. Simplicity: Limit to 5–7 primary items (Miller's Law). Avoid overwhelming users.
  4. Visibility: Active/current page should be highlighted so users know where they are.
  5. Accessibility: Keyboard-navigable, screen-reader friendly, sufficient tap targets on mobile.
  6. Feedback: Hover effects, active states give visual feedback on interaction.
  7. Responsiveness: Hamburger menus or slide-outs on mobile.
  8. Speed: Navigation should load fast — it's the first thing users interact with.
Q3 Discuss website design principles in detail. ▾

Core Website Design Principles

  1. Visual Hierarchy: Guide the user's eye using size, color, and position. Most important content first. Use headings (H1 > H2 > H3), larger text for emphasis, and contrasting colors for CTAs.
  2. The F-Pattern / Z-Pattern: Eye-tracking studies show users scan in F-shape on text-heavy pages and Z-shape on minimal pages. Place key content along these natural scan paths.
  3. Rule of Thirds: Divide the layout into a 3×3 grid and place important elements at intersection points for visual balance.
  4. White Space (Negative Space): Empty space is not wasted space — it improves readability, focuses attention, and conveys elegance.
  5. Color Theory: Use a consistent palette (primary, secondary, accent). Consider color psychology: blue = trust, red = urgency, green = success.
  6. Typography: Pair a display font (headings) with a legible body font. Maintain 16px+ body size, 1.5–1.75 line-height, ~65 chars per line.
  7. Consistency: Same colors, fonts, button styles, icon sets throughout. Creates trust and professionalism.
  8. Mobile-First Design: Design for smallest screens first, then scale up. Ensures usability on all devices.
  9. Fitts's Law: Interactive elements (buttons, links) should be large enough and close enough to be clicked easily, especially on mobile.
  10. Hick's Law: The more choices, the longer decisions take. Minimize options to speed up user decisions.

Q2 Explain different ways to write CSS with example. ▾

CSS (Cascading Style Sheets) can be applied to HTML in three ways:

1. Inline CSS

Styles applied directly on a specific HTML element using the style attribute. Highest specificity but hardest to maintain.

<p style="color: red; font-size: 18px; font-weight: bold;">
  This paragraph is red and bold.
</p>

<h1 style="background: navy; color: white; padding: 10px;">
  Styled Heading
</h1>
HTML

2. Internal (Embedded) CSS

Styles placed inside a <style> tag within the <head> section. Applies to that single HTML document.

<html>
<head>
  <style>
    body {
      background-color: #f0f0f0;
      font-family: Arial, sans-serif;
    }
    h1 { color: darkblue; text-align: center; }
    p  { font-size: 16px; line-height: 1.6; }
  </style>
</head>
<body>
  <h1>Hello World</h1>
  <p>This paragraph uses internal CSS.</p>
</body>
</html>
HTML

3. External CSS

Styles in a separate .css file, linked using <link>. Best practice for large projects — one file styles many pages.

/* styles.css */
body {
  margin: 0;
  padding: 0;
  font-family: Georgia, serif;
  background: #fff;
}

h1 { color: #333; }
.container { max-width: 1100px; margin: 0 auto; }
CSS
<!-- Link in HTML head -->
<head>
  <link rel="stylesheet" href="styles.css">
</head>

Q1 Explain pop-up boxes in JavaScript with example. ▾

JavaScript provides three built-in dialog boxes for interacting with users:

1. alert() — Information Box

Displays a message with an OK button. Used for simple notifications.

alert("Welcome to Web Programming!");
// Returns: undefined
JS

2. confirm() — Confirmation Box

Displays a message with OK and Cancel buttons. Returns true (OK) or false (Cancel).

let result = confirm("Do you want to delete this record?");
if (result === true) {
  alert("Record deleted!");
} else {
  alert("Deletion cancelled.");
}
JS

3. prompt() — Input Box

Displays a message with a text input. Returns the user's input string, or null if cancelled.

let name = prompt("Enter your name:", "Guest");
// 2nd parameter is default value
if (name !== null && name !== "") {
  alert("Hello, " + name + "!");
} else {
  alert("No name entered.");
}
JS

  let original = n;

Q10 Write JavaScript to find whether a number entered by the user is prime or not. ▾

  let n      = parseInt(document.getElementById("num").value);
  let result = document.getElementById("result");

  if (isNaN(n) || n < 1) {
    result.textContent = "Please enter a positive number.";

    result.textContent = n + " is a PRIME number ✅";
    result.style.color = "green";
  } else {
    result.textContent = n + " is NOT a prime number ❌";
    result.style.color = "red";

Q17 Explain callbacks in JavaScript. ▾

A callback is a function that is passed as an argument to another function, and is called (invoked) later — usually when an operation completes or an event occurs.

Callbacks are fundamental to JavaScript's asynchronous programming model.

Simple Callback Example

// Function that accepts a callback
function greet(name, callback) {
  console.log("Hello, " + name);
  callback();  // call the passed function
}

function sayBye() {
  console.log("Goodbye!");
}

greet("Ravi", sayBye);
// Output:
// Hello, Ravi
// Goodbye!
JS

Asynchronous Callback — setTimeout

// Without callback: code runs in order
console.log("Step 1");

setTimeout(function() {   // this function is a CALLBACK
  console.log("Step 2 (after 2s)");
}, 2000);

console.log("Step 3");

// Output order:
// Step 1
// Step 3   ← runs immediately
// Step 2   ← runs after 2 seconds
JS

Callback in Array Methods

let nums = [1, 2, 3, 4, 5];

// forEach uses a callback for each element
nums.forEach(function(n) {
  console.log(n * 2);  // 2, 4, 6, 8, 10
});

// filter uses a callback to test each element
let evens = nums.filter(n => n % 2 === 0);  // [2, 4]
JS
⚠️ Callback Hell: Nesting too many callbacks creates hard-to-read "pyramid" code. Modern JavaScript uses Promises and async/await to solve this.
